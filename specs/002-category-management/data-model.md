# Phase 1 Data Model: Restaurant Category Management (CATEGORY-001)

## Entity: Category

- **Description**: A named grouping that organizes menu items within a single restaurant.
  Owned exclusively by one restaurant tenant.
- **Collection**: `categories`
- **Fields**:
  - `_id`: ObjectId, primary key, auto-generated.
  - `name`: string, required, trimmed, 2–100 characters.
  - `description`: string, optional, trimmed, max 500 characters.
  - `restaurantId`: ObjectId, required, references `restaurants._id`, tenant scope key.
  - `createdAt`: date, auto-generated by Mongoose timestamps.
  - `updatedAt`: date, auto-generated by Mongoose timestamps.
- **Validation rules**:
  - `name`: non-empty after trim, min 2, max 100 characters. Must be unique within `restaurantId`.
  - `description`: optional; if present, max 500 characters after trim.
  - `restaurantId`: always sourced from authenticated token context, never from request body.
- **Indexes**:
  - Unique compound index `{ name: 1, restaurantId: 1 }` — enforces uniqueness at DB level as backup.
  - Index `{ restaurantId: 1 }` — fast paginated list queries per restaurant.
  - Index `{ restaurantId: 1, createdAt: -1 }` — supports default sort by most-recently-created.

## State Transitions

### Category Lifecycle

1. `CREATED` — Admin submits valid create request; category is persisted.
2. `UPDATED` — Admin submits valid update request; name/description changed.
3. `DELETED` — Admin confirms deletion; record removed from collection (hard delete).

No active/inactive states in v1.

## Relationships

- Category → Restaurant: many-to-one. Many categories belong to one restaurant.
- Category → Menu Items (future): one-to-many. Menu items will reference `categoryId`. Cascade behavior is deferred.

## Data Integrity Rules

- All inbound payloads must pass Zod schema validation before any database operation.
- `restaurantId` is never accepted from the request body — always sourced from verified JWT context.
- Name uniqueness must be checked at the application layer (clear `ConflictError`) before hitting the DB unique index.
- Self-update uniqueness check must exclude the document being updated to avoid false conflicts.
